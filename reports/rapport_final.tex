\section{Introduction et Contexte}

\subsection{Présentation du projet}
Ce projet a été réalisé dans le cadre de l'atelier Machine Learning. L'objectif est de développer une solution complète d'analyse comportementale pour une entreprise e-commerce spécialisée dans la vente de cadeaux. À partir d'une base de données clients de 4 372 individus comportant 52 caractéristiques, nous avons construit un modèle capable de prédire le risque de départ (churn) et proposé une segmentation marketing.

\subsection{Objectifs}
\begin{itemize}
    \item Analyser le comportement des clients pour mieux comprendre leurs habitudes d'achat
    \item Prédire le risque de départ des clients (churn)
    \item Développer un modèle de classification performant
    \item Segmenter la clientèle pour des actions marketing ciblées
    \item Créer une interface utilisateur pour la prédiction en temps réel
\end{itemize}

\subsubsection{Valeurs négatives}
Une analyse plus fine a révélé que 44 clients (soit 1\% de la base) présentent un MonetaryTotal négatif. Cela correspond à des cas réels de retours et remboursements. \textbf{Nous avons choisi de conserver ces valeurs} car elles représentent une réalité métier que le modèle doit apprendre à gérer.

\subsection{Traitement des dates}
La colonne RegistrationDate contenait des formats de date inconsistants (ex: "12/03/10", "2010-03-12", "10/18/2010"). Nous avons uniformisé ces formats et converti les dates en timestamps pour les rendre exploitables par le modèle.

\begin{itemize}
    \item Parsing de \texttt{RegistrationDate} avec \texttt{pd.to\_datetime()}
    \item Conversion en timestamp (secondes depuis 1970)
    \item Extraction de nouvelles features : RegYear, RegMonth, RegDay, RegWeekday
\end{itemize}

\subsection{Encodage des variables catégorielles}
Les variables catégorielles (type de client, région, segment RFM, etc.) ont été transformées via un encodage one-hot. Cette opération a considérablement augmenté le nombre de features.

\begin{itemize}
    \item 18 variables catégorielles initiales
    \item \textbf{4 326 features} après One-Hot Encoding
    \item Suppression de \texttt{NewsletterSubscribed} (valeur constante "Yes")
\end{itemize}

\subsection{Normalisation}
Application d'un \texttt{StandardScaler} sur les colonnes numériques (moyenne=0, écart-type=1). Les colonnes booléennes issues de l'encodage one-hot ne sont pas normalisées car elles sont déjà sur une échelle [0,1].

Après toutes ces étapes de préparation, le jeu de données est prêt pour la modélisation.

\end{itemize}

\subsection{Interprétation métier}
\begin{itemize}
    \item \textbf{Recency (9.20\%)} : Un client qui n'a pas acheté depuis plus de 90 jours a un risque de churn multiplié par 3. La stratégie recommandée est d'envoyer une offre personnalisée avant 60 jours d'inactivité.

    \item \textbf{Frequency (5.21\%)} : La fidélisation passe par l'augmentation de la fréquence d'achat via des programmes de fidélité.
\end{itemize}

\section{Résultats et Recommandations}

\subsection{Principaux enseignements}
\begin{enumerate}
    \item La \textbf{date du dernier achat (Recency)} est le facteur le plus prédictif du churn (9.20\%). Plus un client est inactif longtemps, plus le risque augmente.
    \item La \textbf{fréquence d'achat (Frequency)} est le deuxième facteur clé. Les clients qui achètent régulièrement sont naturellement plus fidèles.
    \item Le modèle atteint une précision de \textbf{91.9\%}, ce qui est suffisant pour un déploiement en production.
    \item La segmentation a identifié 4 groupes distincts : 84\% des clients sont occasionnels, représentant un fort potentiel de croissance.
    \item Les clients ayant effectué des retours (ReturnRatio élevé) présentent un risque de churn plus important.
\end{enumerate}

\subsection{Recommandations marketing}
\begin{itemize}
    \item \textbf{Cibler les clients à risque élevé} : Envoyer des offres personnalisées aux clients identifiés avec probabilité > 70\%
    \item \textbf{Réactiver les clients dormants} : Campagne d'emailing ciblée après 60 jours d'inactivité
    \item \textbf{Fidéliser les clients VIP} : Programme de fidélité avec avantages exclusifs
    \item \textbf{Suivi des clients occasionnels} : Newsletter engageante, recommandations produits personnalisées
\end{itemize}

\section{Limites et Perspectives}

\subsection{Limites du modèle}
Malgré de bonnes performances, notre solution présente certaines limites :
\begin{itemize}
    \item \textbf{Données limitées} : 4 372 clients seulement, ce qui limite la généralisation du modèle à d'autres populations
    \item \textbf{Features manquantes} : Absence de données comportementales web (pages vues, temps passé sur le site)
    \item \textbf{Seuil statique} : Le seuil de décision (0.5) n'est pas optimisé pour le coût métier (un faux négatif coûte plus cher qu'un faux positif)
    \item \textbf{Application simplifiée} : Seules 4 features sont utilisées dans l'interface par souci de simplicité
    \item \textbf{Déséquilibre des segments} : Certains segments sont très petits (2 clients pour le cluster 1)
\end{itemize}

\subsection{Perspectives d'amélioration}
Plusieurs pistes pourraient être explorées pour améliorer la solution :
\begin{itemize}
    \item \textbf{Plus de données} : Intégrer les logs de navigation et l'historique des emails pour enrichir les features
    \item \textbf{Optimisation du seuil} : Ajuster le seuil de décision selon le coût d'acquisition client (CPA)
    \item \textbf{Modèle temporel} : Utiliser des séries temporelles (LSTM) pour détecter les changements de comportement dans le temps
    \item \textbf{API complète} : Ajouter toutes les features dans l'API de prédiction pour des résultats plus précis
    \item \textbf{Déploiement cloud} : Mettre l'application sur AWS, Heroku ou Render pour un accès en ligne
    \item \textbf{Test autres algorithmes} : XGBoost, LightGBM, ou des réseaux de neurones pour comparer les performances
\end{itemize}

\section{Conclusion}

Ce projet a permis de :
\begin{itemize}
    \item Maîtriser l'ensemble de la chaîne de traitement en data science (exploration → préparation → modélisation → évaluation → déploiement)
    \item Appliquer des techniques avancées de prétraitement (imputation par médiane, encodage one-hot, normalisation StandardScaler)
    \item Développer un modèle de classification performant (91.9\% d'exactitude)
    \item Créer une segmentation client actionable avec K-Means (4 segments identifiés)
    \item Déployer une application web fonctionnelle avec Flask
\end{itemize}

Les résultats obtenus démontrent la faisabilité d'une solution de prédiction du churn client, permettant à l'entreprise d'anticiper les départs et d'optimiser ses stratégies marketing. La segmentation a également mis en évidence que 84\% des clients sont occasionnels, représentant un fort potentiel de croissance pour l'entreprise.
